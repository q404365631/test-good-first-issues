# Contributing to Test Good First Issues Repository

Thank you for your interest in contributing! This repository is designed specifically to help newcomers learn the open-source contribution workflow.

## Code of Conduct

Please treat everyone with respect and kindness. We welcome all contributors regardless of experience level.

## How to Contribute

### Finding Your First Issue

Look for issues labeled `good first issue` - these are perfect for beginners! They're designed to be small, self-contained tasks that will help you understand:

- How to read and understand code
- The pull request process
- Code review feedback
- Git workflows

### Reporting Bugs

If you find a bug, please create an issue with:
- A clear title describing the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment (browser, OS, etc.)

### Suggesting Enhancements

For feature suggestions:
- Explain the problem it solves
- Describe the solution you'd like
- Provide examples if helpful
- Consider alternatives you've explored

## Getting Started Locally

1. Fork this repository on GitHub
2. Clone your fork locally:
   ```bash
   git clone https://github.com/your-username/test-good-first-issues.git
   cd test-good-first-issues
   ```

3. Create a branch for your changes:
   ```bash
   git checkout -b my-contribution
   ```

4. Make your changes

5. Commit your changes:
   ```bash
   git add .
   git commit -m "Add a clear, descriptive message"
   ```

6. Push to your fork:
   ```bash
   git push origin my-contribution
   ```

7. Open a Pull Request on GitHub

## Development Guidelines

### Commit Message Style

Use present tense and imperative mood:
- ✅ `Add README with setup instructions`
- ✅ `Fix typo in CONTRIBUTING.md`
- ❌ `Added README` or `Adding feature`

### Code Quality

- Keep changes focused and small
- Follow existing patterns in the codebase
- Add comments when something might not be obvious

## Review Process

All pull requests will be reviewed by maintainers. Be patient - this is part of learning!

You'll receive feedback that will help you grow as a developer. Don't take criticism personally - it's about improving the code!

## Questions?

Feel free to ask questions in issues. The community is here to help you succeed!
